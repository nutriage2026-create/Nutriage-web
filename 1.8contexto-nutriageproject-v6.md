## Stack tecnológico
- **100% HTML + CSS + JS** en un solo archivo (sin frameworks, sin backend)
- **localStorage** como base de datos del navegador (clave: `nutriageV3`)
- **Jitsi Meet** integrado vía `external_api.js` para videollamadas (gratuito, sin instalar nada)
- **Sin servidor, sin base de datos externa** — todo corre en el navegador
- ⚠️ Jitsi solo funciona desde servidor web (no doble clic). Usar **VS Code Live Server** o subir a **Netlify**

## Arquitectura del sistema — 3 vistas

### 1. Vista Paciente (pública) — DUAL SUBSISTEMA
La vista paciente tiene **dos pestañas** con tabs táctiles grandes (mobile-first):

**Pestaña A — "📅 Nueva reserva"** (flujo de 4 pasos):
1. **Calendario** → elige fecha y hora disponible
2. **Formulario** → 33 campos clínicos
3. **Confirmación** → resumen de reserva
4. **Pago** → transferencia bancaria + subir comprobante
- Al finalizar se genera un **número de ticket único** (ej: `NA-20260409-001`) que el paciente debe guardar

**Pestaña B — "🎟️ Ya tengo ticket"**:
- Campo de texto grande para ingresar el número de ticket
- Si el ticket existe y la cita es HOY → muestra botón verde "Unirme a la videollamada"
- Si la cita es otro día → mensaje informativo con la fecha
- Si el ticket no existe → mensaje de error

### 2. Vista Nutricionista (contraseña: fernanda2026)
- **Citas del día**: nombre, hora, motivo, número de ticket badge, botón "Sala Virtual" (lanza Jitsi)
- **Calendario editable** de disponibilidad (activa/desactiva días y horarios)
- **Registros de pacientes** desplegables con ficha resumida y ticket visible
- **Ficha clínica completa** por paciente (modal con TODA la info)
- **Sección Estadísticas de Ingresos**:
  - 4 tarjetas: ingresos hoy, semana, mes, consultas semanales
  - Gráfico de barras SVG (sin librerías) — 7 días o 30 días con etiquetas rotadas
  - Fernanda NO sabe que el admin también tiene este gráfico

### 3. Vista Administrador (contraseña: admin2026)
- **Tarjetas de estadísticas**: total pacientes, consultas hoy, ingresos esperados, formularios con comprobante, nº nutricionistas, ingresos anuales estimados
- **Tabla de pacientes**: columna Ticket + Nombre + Email + Teléfono + Fecha + Hora + Motivo + Comprobante + Detalle
- **Botón "💾 Descargar Excel"**: genera CSV con BOM UTF-8 con: Ticket, Nombre, Email, Teléfono, RUT/ID, Fecha, Hora, Motivo, Estado, Comprobante
- **Ficha clínica completa** por paciente (modal)
- **Estadísticas — Fernanda Ugarte**: tabla sesiones e ingresos por período
- **Gráfico de torta/donut**: % de pacientes por nutricionista (preparado para múltiples nutris)
- **Gráfico de barras de ingresos** por día: 7 días o 30 días (mismo motor que el de nutricionista)

## Sistema de Tickets
- Formato: `NA-AAAAMMDD-NNN` → ej: `NA-20260409-001`
- Se genera automáticamente al finalizar la reserva (función `generarTicket(dateKey)`)
- Secuencial por día: el primer paciente del 9 de abril es `001`, el segundo `002`, etc.
- Se guarda como propiedad `ticket` en el objeto paciente en localStorage
- Aparece como badge morado en: citas del día (nutri), registros (nutri), tabla admin

---

## Sistema de Videollamadas (Jitsi Meet)
- Librería: `https://meet.jit.si/external_api.js` cargada en `<head>`
- Nombre de sala generado automáticamente: `NutriAge_` + teléfono limpio del paciente
- Fernanda entra con toolbar completo (micro, cámara, pantalla, chat, participantes)
- El paciente entra con toolbar reducido (micro, cámara, chat, colgar)
- El paciente entra con micro muteado por defecto
- Modal full-screen con header NutriAge (logo, nombre paciente, indicador "En llamada")
- Funciones clave: `iniciarLlamada(seed, nombre)`, `unirseComoP()`, `cerrarLlamada()`
- Config: `prejoinConfig:{enabled:false}` + `prejoinPageEnabled:false` para entrar directo
- Fallback de 3.5s para ocultar spinner si los eventos no disparan

## Campos del formulario (33 campos)
**Datos personales:** nombre, email, teléfono, edad, ocupación, peso (kg), talla (metros)

**Historial médico:** motivo de consulta, enfermedades diagnosticadas, lesiones recientes, antecedentes familiares, cirugías

**Estilo de vida:** convivencia/quién cocina, actividad física, horario laboral, horario de sueño

**Hábitos alimentarios:** alimentos que no le gustan, almuerzo frecuente, horarios de comida, alergias/intolerancias, suplementos

**Consumo:** alcohol, tabaco/drogas

**Digestión:** síntomas gastrointestinales

**Frecuencia semanal:** cereales, legumbres, pescado, fruta, verduras, dulces, lácteos

**Registro 24h:** descripción libre

## Diseño y estilo
- **Paleta:** morado profundo `#3d2459` + verde salvia `#4a8c54` + crema `#faf7f2`
- **Variables CSS:** `--p1` al `--p6` (morados), `--s1` al `--s5` (verdes), `--g1` al `--g5` (grises)
- **Tipografía:** Cormorant Garamond (títulos) + Nunito (texto)
- **Tono:** natural, alimentación saludable, amigable, femenino
- **Mobile-first:** bottom nav en celular (3 tabs), nav superior en desktop
- Fuente `16px!important` en inputs para evitar zoom en iOS
- Step bar en scroll horizontal en mobile (`step-bar-wrap` con `overflow-x:auto`)
- Tabs de paciente con `min-height:52px` para fácil toque en mobile
- Modal Jitsi `position:fixed;inset:0;z-index:9999`

## Funciones JS clave
| Función | Descripción |
|---|---|
| `getDB()` / `saveDB(db)` | Lee/escribe localStorage (`nutriageV3`) |
| `dk(date)` | Convierte Date a string `YYYY-MM-DD` |
| `fmtMoney(n)` | Formatea número como `$15.000` CLP |
| `generarTicket(dateKey)` | Genera código `NA-AAAAMMDD-NNN` |
| `loginConTicket()` | Valida ticket e ingresa al subsistema B |
| `switchPatientMode(mode)` | Alterna entre `'nueva'` y `'ticket'` |
| `renderNutriAppts()` | Renderiza citas del día con tickets y botón sala |
| `renderNutriRecords()` | Renderiza lista de pacientes con fichas |
| `renderNutriStats()` | Calcula ingresos y dibuja gráfico de barras (nutri) |
| `renderAdmin()` | Renderiza toda la vista admin |
| `renderAdminPieChart()` | Dibuja gráfico donut de pacientes por nutricionista |
| `buildBarSvg(svgId, days, nutri)` | Motor SVG para gráficos de barras (7 o 30 días) |
| `exportarExcel()` | Genera CSV descargable con todos los pacientes |
| `iniciarLlamada(seed, nombre)` | Abre sala Jitsi desde panel nutricionista |
| `unirseComoP()` | Paciente se une a sala Jitsi |
| `cerrarLlamada()` | Cierra y limpia instancia Jitsi |
